Retail Sales Intelligence — Power BI Dashboard

GitHub Live Demo

Overview

Dashboard Power BI dua halaman yang mengubah lebih dari 23.000 transaksi retail menjadi keputusan bisnis yang jelas. Halaman Overview menampilkan performa revenue keseluruhan, tren bulanan, distribusi geografis, dan produk terlaris. Halaman Customer & Retur membedah siapa pelanggan terbaik, di mana retur paling banyak terjadi, dan bagaimana polanya berubah dari waktu ke waktu.

Problem

Kebanyakan bisnis retail punya ribuan transaksi setiap bulan — tapi datanya tersimpan di spreadsheet terpisah, tidak ada yang tahu produk mana yang paling sering diretur, pelanggan mana yang paling bernilai, atau bulan mana yang paling lemah. Keputusan dibuat berdasarkan feeling, bukan angka. Dan selama itu, uang terus bocor diam-diam.

Value Proposition

Revenue Visibility

Satu tampilan tunggal untuk £9.79M revenue dari 23K transaksi — dipilah per bulan, per negara, per produk, dalam hitungan detik.

Return Intelligence

Menemukan £478K yang "hilang" akibat retur 5.13% — siapa yang retur, dari mana, dan kapan polanya terjadi.

Customer Intelligence

Top 10 pelanggan menyumbang £1.37M. 93.92% transaksi dari guest, bukan member — peluang loyalty program yang nyata.

Sinok Homestay — Booking & Forecasting System

GitHub Live Demo

Overview

Sinok Homestay adalah project simulasi sistem data dan otomasi untuk bisnis homestay. Project ini dibuat untuk menjawab masalah umum pada usaha penginapan kecil: data booking masih tersebar di chat, buku catatan, dan beberapa platform OTA, sehingga sulit dipantau secara cepat dan rawan terjadi double booking.

Melalui sistem ini, booking dapat dicatat lewat Telegram Bot, tersimpan otomatis ke Google Sheets, memicu reminder update OTA, lalu divisualisasikan dalam dashboard okupansi dan revenue. Data historis juga digunakan untuk membuat proyeksi okupansi dan pendapatan bulan berikutnya.

Problem

Banyak usaha homestay memiliki data operasional yang sebenarnya kaya insight, tetapi belum dimanfaatkan karena pencatatan masih manual. Admin sering menerima booking dari berbagai channel seperti Traveloka, Agoda, Booking.com, atau direct order, namun data tersebut tidak selalu tercatat dalam satu sistem yang rapi.

Akibatnya, pemilik sulit mengetahui performa okupansi, channel mana yang paling efektif, kapan periode ramai/sepi, serta berapa potensi revenue bulan berikutnya. Risiko lain yang muncul adalah keterlambatan update ketersediaan kamar di OTA, sehingga peluang double booking menjadi lebih besar.

Solution

Project ini membangun alur sederhana namun end-to-end: admin cukup memasukkan booking melalui format command di Telegram Bot. Data booking kemudian divalidasi, disimpan ke Google Sheets, dan langsung memicu pesan pengingat agar admin memperbarui ketersediaan kamar di semua OTA.

Data yang terkumpul digunakan untuk membuat dashboard analisis okupansi, revenue, sumber booking, rata-rata lama tinggal, harga per malam, serta tren musiman. Selain itu, sistem juga menghitung proyeksi okupansi dan revenue untuk bulan berikutnya berdasarkan pola historis.

Workflow

Telegram Bot→ n8n Workflow→ Google Sheets→ Dashboard→ Forecast
  1. Admin mengirim command booking melalui Telegram.
  2. Bot memvalidasi format input, tipe kamar, tanggal, channel, dan nominal booking.
  3. Data booking otomatis masuk ke Google Sheets.
  4. Sistem mengirim reminder agar admin memperbarui ketersediaan kamar di OTA.
  5. Dashboard membaca data booking untuk menampilkan performa okupansi dan revenue.
  6. Forecast menghitung estimasi okupansi dan pendapatan bulan berikutnya.

Value Proposition

Visibility

Dashboard mengubah data booking mentah menjadi informasi yang mudah dibaca: tren okupansi, revenue, channel booking, harga, dan pola musiman.

Otomasi

Booking baru bisa dicatat lewat satu baris command chat dan langsung tersimpan ke database, tanpa perlu membuka laptop atau menginput manual berkali-kali.

Foresight

Data historis digunakan untuk memperkirakan okupansi dan revenue bulan berikutnya, sehingga pemilik homestay dapat mengambil keputusan lebih cepat.

Pengembangan berikutnya diarahkan pada perhitungan sisa kamar secara real-time, validasi ketersediaan berdasarkan tanggal check-in/check-out, serta forecast yang lebih adaptif terhadap musim liburan dan pola pemesanan mendadak.
